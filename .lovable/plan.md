

# Fix Email Confirmation Links to Use Custom Domain

## Problem
The `confirmationUrl` in auth emails comes from `payload.data.url`, which is generated by the auth system using the project's default Site URL (the Lovable preview URL). Even though `siteUrl` was updated, the actual clickable links (confirm account, reset password, etc.) still point to the Lovable domain.

## Solution
Rewrite `payload.data.url` in the `auth-email-hook` edge function to replace the Lovable project URL with `https://www.getheavencoin.com` before passing it to templates.

### File: `supabase/functions/auth-email-hook/index.ts`

Add a URL rewriting step at line ~225, before building `templateProps`:

```typescript
// Rewrite confirmation URL to use custom domain
const CUSTOM_SITE_URL = `https://www.${ROOT_DOMAIN}`
const rawUrl = payload.data.url || ''
const confirmationUrl = rawUrl.replace(
  /https?:\/\/[^/]*\.lovable\.app/,
  CUSTOM_SITE_URL
)
```

Then use this rewritten `confirmationUrl` in `templateProps` instead of `payload.data.url`.

### Deploy
Redeploy `auth-email-hook` edge function to apply the fix.
